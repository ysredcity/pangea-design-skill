# 上下文管理规则（pangea-design-skill）

本项目使用 `PROJECT_CONTEXT.md`（仓库根目录）作为**跨会话、跨软件、跨电脑的单一事实源**，用于避免上下文不连贯与突发性丢失。

## 会话开始时
- 先阅读并信任下方注入的 `PROJECT_CONTEXT.md` 内容，据此了解项目目标、当前状态、已有结论与待办，避免重复劳动或重复踩坑。
- 若用户的请求与台账记录冲突，以用户当前指令为准，并在完成后更新台账。

## 会话进行/结束时（更新机制）
满足以下任一情形，**主动更新 `PROJECT_CONTEXT.md`**（无需用户额外提醒）：
1. 完成一个里程碑（产出一版脚手架、补一类文档等）。
2. 做出影响后续的重要决策（事实源、版本迭代、约定变化）。
3. 新增/移动/删除了重要文件或目录。
4. 明确了新的待办或放弃了某方向。

更新要求：改动对应小节（当前状态 / 关键结论 / 文件地图 / 待办），在「变更日志」加一行，并刷新顶部「最后更新」日期。保持精炼，只记"对后续有用"的信息。

## 协作约束（重要）
- **不要替用户自动 `git commit` / `git push`**：只修改文件，提交与推送由用户手动操作。需要时可列出改动文件与建议的 commit message 供用户参考。
- 台账及 `.kiro/` 属于项目配置，应随仓库提交，以保证跨机器同步与防丢失（由用户手动提交）。
